Wikipedia Scraper

One of the main functions of Localify.org is to recommend artists that are in a users local area. To do this, we need to know an artists origin to make an artist to city relationship within our database. One of the best sources for an artist origin is Wikipedia. As seen with the Red Hot Chili Pepper's Wikipedia page, their origin is listed on the sidebar. One of my main contributions to the Localify project was creating a Wikipedia scraper that would try to find the origin of each artists in our database.

    In order to accomplish this, I created a python script. Here is a outline of how the script works:
  • Make a list of all artists in our database that have not been searched on wikipedia
  • For each artist
    • Find the corresponding wikipedia page
    • Try to access wikipedia.org/artist_name
    • Confirm that the wikipedia page is for a musician (search for keywords Ex: "discography")
    • If not found, try again by adding extensions (Ex: " (band)") to the wikipedia url
  • Search the wikipedia page for information about origin or birthplace
    • This was done using Python package Beautiful Soup
  • If origin/birthplace is found, find the corresponding city in our database
  • Create a relation between artist and city in our database
    • Want to see the python code behind this? Check it out! The scraper begins with the run(self) function. Now that our database has more connections between artists and locations, our system can recommend more local artists to our users!
